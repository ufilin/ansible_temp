# Ansible роль: vector-role

**License**: MIT

Роль для автоматической установки и настройки [Vector](https://vector.dev/) — высокопроизводительного сборщика и маршрутизатора логов.

## Описание

Данная роль выполняет следующие действия:

- Устанавливает DNS-сервер (8.8.8.8) в `/etc/resolv.conf`.
- Устанавливает необходимые зависимости (`curl`, `tar`).
- Скачивает и распаковывает архив Vector с GitHub.
- Копирует бинарный файл Vector в `/usr/bin/vector`.
- Создаёт каталог конфигурации `/etc/vector`.
- Генерирует конфигурационный файл Vector из шаблона `vector.yml.j2`.
- Создаёт systemd-сервис Vector из шаблона `vector.service.j2`.
- Включает и запускает сервис.

Роль предназначена для целевых систем на базе Debian/Ubuntu (используется модуль `apt`).

## Требования

- Ansible версии 2.9 или выше.
- Управляемый хост с ОС семейства Debian/Ubuntu.
- Доступ к интернету для загрузки архива Vector.
- Права `become: true` для установки пакетов и создания системных файлов.

## Переменные

Роль использует следующие переменные (задаются в `defaults/main.yml` и `vars/main.yml`):

| Имя переменной      | Значение по умолчанию   | Описание                          |
|---------------------|-------------------------|-----------------------------------|
| `vector_version`    | `"0.21.1"`              | Версия Vector для установки       |
| `vector_config_dir` | `"/etc/vector"`         | Путь к каталогу конфигурации      |

Эти переменные можно переопределить в playbook или инвентори.

## Шаблоны

Роль использует два шаблона Jinja2:

- **`vector.yml.j2`** — конфигурационный файл Vector (`/etc/vector/vector.yaml`).  
- **`vector.service.j2`** — unit-файл systemd для Vector.
